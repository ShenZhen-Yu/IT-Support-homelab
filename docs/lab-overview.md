# Lab Overview

## Purpose
This lab was built to practice hands-on IT support and system administration skills in a small virtual environment. The goal is to simulate a basic business-style setup using a Windows Server VM and a Windows Client VM.

## Current Lab Components
- 1 Windows Server virtual machine
- 1 Windows Client virtual machine
- Static IP addressing
- Basic network connectivity testing

## Why I Built This Lab
I built this lab to gain hands-on practice with:
- Virtual machine deployment
- Windows networking
- Basic troubleshooting
- Active Directory administration
- Help desk style workflows

## Lab Roles
### Server VM
The server VM is intended to be used as the main infrastructure system. In the next phase, it will be promoted to a Domain Controller and used for Active Directory and DNS.

### Client VM
The client VM is used to simulate an end-user workstation. It will later be joined to the domain for login, user account, and policy testing.

## Current Status
At this stage, both VMs have been created and configured with static IP addresses. Basic connectivity between the systems has been tested successfully after resolving a firewall-related issue.

## Planned Expansion
The next phase of the lab will include:
- DNS configuration
- Domain join
- User and group management
- Organizational Units
- Group Policy
- IT support scenarios such as password resets and account unlocks
